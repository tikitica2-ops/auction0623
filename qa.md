# 🔍 QA 담당자 테스트 및 품질 검증서 - qa.md

* **호출 명령어:** `/qa`
* **주요 역할:** 시스템 통합 테스트 계획 수립, 프롬프트 환각(Hallucination) 억제율 검증, 에이전트 라우팅 분기 정확성 검정, 경매일기 JSON 반환 포맷 유효성 검사

---

## 1. 테스트 기본 전략 및 중점 검증항목
본 프로젝트는 제한시간 내 시연을 수행해야 하는 발표 중심의 프로젝트이므로, **"실패하지 않는 안정적인 구동"**이 최우선 테스트 목표입니다.

1. **에이전트 라우팅 정확도 (100% 분기 타겟):** 절차 질문 ➔ 절차 에이전트, 개념 질문 ➔ 튜터 에이전트로 완벽히 라우팅되는지 확인합니다.
2. **사례 퀴즈 채점의 공정성 및 정합성:** 정답 키와 학생 답안을 비교 분석할 때 평가자 LLM이 허황된 채점(환각)을 내리지 않는지 검증합니다.
3. **경매일기 JSON 포맷 검증:** Gemini가 작성하는 분석 일기가 JSON 형태를 엄격하게 지키며 Streamlit UI가 깨지지 않는지 상시 파싱 테스트를 거칩니다.

## 2. 핵심 QA 테스트 시나리오 및 체크시트

### 2.1 시나리오 A: 에이전트 라우터 및 Q&A 테스트
- **테스트 케이스 A-1: 절차 질문**
  - **입력:** "경매 낙찰받고 대금 납부 기한은 언제까지인가요?"
  - **예상 경로:** `[라우팅: procedure]` (절차 안내 에이전트)
  - **검증항목:** 답변 하단에 민사집행법 제142조 등 법률적 출처가 팝업 형태로 함께 제시되는지 확인.
- **테스트 케이스 A-2: 용어/개념 질문**
  - **입력:** "유치권이 대체 뭐야? 쉽게 가르쳐줘."
  - **예상 경로:** `[라우팅: tutor]` (권리분석 튜터 에이전트)
  - **검증항목:** 지식베이스 `glossary.json`의 유치권 해설 정보가 RAG로 검색되어 답변에 반영되는지 확인.

### 2.2 시나리오 B: 권리분석 퀴즈 채점 루프 테스트 (Evaluator-Optimizer)
- **테스트 케이스 B-1: 완전 정답 답안 채점**
  - **입력:** 등기부 퀴즈 사례 1번에 대해 정답에 근접한 서술형 답안 제출
  - **예상 스코어:** 3점 ~ 4점 (4점 만점)
  - **검증항목:** 부분 점수 항목(말소기준권리, 대항력 판정, 인수 권리, 위험도)이 개별 피드백과 매칭되는지 확인.
- **테스트 케이스 B-2: 오답 답안 채점**
  - **입력:** "대항력 무조건 있습니다. 말소기준권리는 2026년입니다." (사실과 전혀 다른 엉뚱한 답안)
  - **예상 스코어:** 0점 ~ 1점
  - **검증항목:** 오답 처리가 정상적으로 이루어지고, 학습 가이드가 올바르게 피드백 총평에 제공되는지 확인.

### 2.3 시나리오 C: 경매 학습일기 분석 테스트 (Google Gemini API)
- **테스트 케이스 C-1: 개념 오류 발견 및 지적**
  - **입력:** "오늘 사례 1번 아파트 분석을 했는데, 전입신고일이 2024년 5월이고 저당권 일자가 2023년 4월이라서 임차인은 대항력이 존재하고 안전하다." (대항력 부존재 상태를 존재한다고 오판한 경우)
  - **예상 출력:** 
    - `emotional_support`: 학습을 응원하는 격려 메시지
    - `risk_flags`: 임차인 전입일이 저당권보다 늦어 대항력이 없음을 따끔히 짚어주는 위험 체크 리스트
    - `recommended_study`: '대항력 요건' 및 '말소기준권리' 용어 공부 권장 가이드
  - **검증항목:** Gemini가 출력한 로우 텍스트가 Streamlit 내에서 JSON 구조체로 정상 파싱되어 예쁜 UI 카드로 레이아웃이 터지지 않고 표시되는지 점검.

## 3. 회귀 테스트 및 예외 처리 지침 (CTO 연계)
- **API Timeout 대비:** 외부 API(OpenAI, Gemini) 통신 중 타임아웃이나 할당량 부족 에러가 날 경우 UI 전체가 멈추지 않고, `"네트워크가 다소 혼잡하여 답변이 지연되고 있습니다. 잠시 후 시도해주세요."`라는 팝업 토스트 알림을 띄우는지 점검합니다.
- **DB 컬렉션 유효성:** 최초 가동 시 ChromaDB 내 컬렉션이 없으면 `verify_and_build_db()`가 동작하여 자동으로 RAG DB를 빌드 및 복구하는지 유효성 검사를 상시 수행합니다.
